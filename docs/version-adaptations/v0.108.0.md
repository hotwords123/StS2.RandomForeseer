# StS2 v0.108.0 adaptation summary

对比范围：游戏源码从 v0.107.1 `cd14164` 到 v0.108.0 `432ae91`。

## 已适配内容

- `CardCreationOptions` 移除了 `CustomCardPool` / `WithCustomPool` 形态，预测侧改为 `CardPools` + `CardPoolFilter`，并在 clone 时保留 `Source`、`RarityOdds`、`CardPoolFilter`、`Flags` 和 `RngOverride`。
- 战斗卡牌奖励现在带 `CardCreationFlags.IsFromCombat`；`LastingCandy` 从 `CombatsSeen` / `AfterCombatEnd` 改为 `CombatRewardsSeen` / `BeforeCombatRewardOffered`，预测镜像已同步新触发条件。
- `PotionRewardOdds.Roll` 移除了 `AscensionManager` 参数；强制药水奖励在 v0.108.0 中不再消耗 odds RNG。预测仍保留 `GoldReward.Populate` 的 rewards RNG 步进，并更新了注释说明。
- `MassiveScroll` 由自定义卡表改为角色卡池 + 无色卡池 + 多人卡过滤器，拾取预测已同步。
- 伤害 API 新增 `CardPlay?` 参数；hover/preview 预测沿用原版预览形态，向 `Hook.ModifyDamage` 和 `AttackCommand.FromCard` 传 `null`，并在相关位置标注原因。
- `FrostOrb` 在 `HibernatePower` 存在时会让所有玩家获得被动/激发格挡；`OrbBehavior` 已同步 owner-first 格挡顺序和激发 target list。
- `PunchOff` 战斗的额外遗物 + 药水在 v0.108.0 改为仅事件 owner 获得。预测保留正常战斗奖励的逐玩家快进，只在当前 owner 的正常卡牌奖励后预测 owner-only 额外奖励。
- `BattlewornDummy` 拆成 V1/V2/V3 no-reward encounter，奖励移到 `Resume` 中通过自定义奖励/升级发放；当前 Setting 2 预测已补充版本注释并保留战斗创建前的一次固定 HP Niche 步进。
- Mod loader 从单 assembly 改为多 assembly；hook 反射识别逻辑使用 v0.108.0 新增的 `AssemblyInfo.ModMap`，并补充版本注释。
- hook mirror 文档已同步 draw、exhaust、damage given/received、damage modifier、card reward、orb、end-turn 和 overview 的 v0.108.0 差异。
