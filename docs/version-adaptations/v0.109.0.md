# StS2 v0.109.0 adaptation summary

对比范围：游戏源码从 v0.108.0 `432ae91` 到 v0.109.0 `3ca1650`。

## 已适配内容

- `Rng` 从 seed/counter 重建改为序列化完整 Xoshiro 状态；预测克隆现在通过
  `ToSerializable()` / `Rng(SerializableRng)` 完成，固定步进通过丢弃克隆流的 raw draw
  实现，不再访问已移除的 `Seed`、`Counter` 或 `FastForwardCounter`。
- 卡牌结算位置从 `(PileType, CardPilePosition)` 扩展为 `CardLocation`。战斗模拟器现在自行
  mirror 基础结果位置以及 `ParticleWall`、`ShiningStrike`、`TheBall` 的 override；`TheBall`
  使用克隆的 `CombatTargets` / `Shuffle` RNG，并保留原始出牌者、跨玩家转交和目标牌堆
  顺序，避免在脱离真实牌堆的预测牌上调用会推进真实 RNG 的原版 virtual 方法。
- `CardPlay.Player` 已贯穿模拟出牌，Ripple Basin 等历史归属判断改用原始出牌玩家而不是
  结算后可能变化的 `Card.Owner`。
- 多人战斗中的 AnyPlayer 药水预测显式区分药水 owner 与效果 target。随机印牌药水使用
  目标玩家的卡池和 `CombatCardGeneration`；抽牌、Glowwater Potion、Distilled Chaos 和
  Entropic Brew 使用目标玩家的牌堆、药水槽和 RNG。目标选择期间的提示随当前玩家目标
  刷新，并在取消、完成或会话替换时清理事件和 UI 状态。
- 新增 `AfterBlockBroken(PlayerChoiceContext, target, breaker)` mirror：Hand Drill 在条件
  命中时保留 Apply Power 风险，Burrowed Power 的移除/怪物状态效果按当前预测边界忽略；
  dispatch 顺序和致死破甲时的无 guard listener iteration 与原版一致。
- `PillarOfCreationPower` 已同步每回合仅第一张生成牌触发，`SoulboundPower` 已同步随机插入
  抽牌堆并消耗克隆 Shuffle RNG，`UnderworldPower` 已增加同阵营门控；Diamond Diadem 的
  旧回合末注册随原版重做移除。
- Sere Talon / Distinguished Cape 的拾取效果在 v0.109.0 中交换；随机诅咒预测已移动到
  Distinguished Cape，Sere Talon 不再错误读取已移除的 `Curses` dynamic var 或推进 Niche RNG。
- 新卡 Abundance（富足）已接入战斗随机印牌预测，展示 3 张不重复的角色能力牌候选，并在
  卡牌升级时展示升级后的候选。
- mirror model provenance 改用 v0.109.0 `AssemblyInfo.ModForType`，兼容 base game、mod 和测试
  mock type 的统一识别。
- One for All 的 X 费排除、Vulnerable 对攻击宠物继承 Cruelty、Toy Box 数量和事件候选池等
  由原版 value hook、canonical model 或动态 `AllPossibleOptions` 自动继承；相关 hook 文档已同步。
